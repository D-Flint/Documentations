Getting Started with Cline CLI
==============================

What is Cline?
--------------
Cline is an AI-powered coding assistant that lives inside your IDE (VS Code,
Cursor, or Windsurf). It can understand your codebase, execute terminal
commands, edit files, and automate complex development workflows.

Installation
------------

1. Open VS Code (or your supported IDE).
2. Go to the Extensions marketplace (Ctrl+Shift+X).
3. Search for "Cline" by Saoud Rizwan.
4. Click Install.
5. Once installed, you'll see the Cline icon in the activity bar.

Configuration
-------------

After installation, Cline needs an API provider to function:

1. Click the Cline icon in the activity bar.
2. Select your preferred API provider:
   - OpenAI (GPT-4, GPT-4o, etc.)
   - Anthropic (Claude 3.5 Sonnet, Claude 3 Opus)
   - Google Gemini
   - OpenRouter (access to multiple models)
   - Local models via Ollama or LM Studio
3. Enter your API key for the chosen provider.
4. Optionally configure custom instructions to shape Cline's behavior.

Basic Usage
-----------

Chat with Cline:
- Press Ctrl+Shift+I (or Cmd+Shift+I on macOS) to open Cline.
- Type your request in natural language.
- Cline will analyze your request and act accordingly.

What Cline Can Do:
- Read and edit files in your project
- Execute terminal commands
- Search and navigate your codebase
- Create new files and folders
- Install dependencies and run builds
- Debug errors and fix issues
- Generate unit tests and documentation
- Refactor code across multiple files

Advanced Features
-----------------

Custom Modes:
  Cline supports custom modes (Code, Architect, Ask) that change how it
  interacts with your project. Configure these in the Cline settings.

Slash Commands:
  Use "/" in chat to trigger special commands like:
    /help      -- Show available commands
    /clear     -- Clear the conversation
    /task      -- Start a new task context

Checkpoints:
  Cline automatically creates checkpoint snapshots before making changes,
  allowing you to review and revert edits if needed.

Web Search:
  Cline can search the web for documentation, APIs, or solutions when
  given the appropriate tool permissions.

Best Practices
--------------

- Be specific in your requests (include error messages, file paths, etc.)
- Use the custom instructions to set coding style and conventions
- Review changes before accepting them
- Leverage checkpoints to safely experiment
- Pair Cline with version control (git) for full traceability

Tips
----
- Start with small tasks to understand how Cline works
- Use descriptive prompts for better results
- Enable web search for up-to-date library documentation
- Keep your API key secure and never commit it to version control
